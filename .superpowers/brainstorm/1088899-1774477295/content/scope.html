What should the Agent Builder do in 5.0?

4.0 had a full 6-step wizard with LLM-compiled blueprints, sub-agent spawning, SQLite persistence, launch modes, approval policies, and output modes. What do we rebuild?

A

Clean rebuild of the 4.0 core

Recreate the essential agent builder: goal input → LLM compiles blueprint → user reviews → save & run. Sidebar list with status. Run history. Clean, no bloat — but fully functional end-to-end.

B

Simpler "prompt + tools" agent

Skip the blueprint wizard. Just: name your agent, write a system prompt, pick which tools it can use, save it. Run it from the sidebar. Lean, direct, no LLM compilation step.

C

Goal-driven with CLI superpower

Like A but with CLI/terminal as a first-class tool. Agent can run shell commands, scripts, watch files. Showcases a "devops/coding agent" use case as the demo. The robust use case you mentioned.
